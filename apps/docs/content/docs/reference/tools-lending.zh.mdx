---
title: "工具：借贷"
description: 通过 Aave、Compound、Kamino 和 Marginfi 进行供给、提款、借款、还款和读取借贷仓位。
---

借贷工具管理跨多个 DeFi 协议的仓位。所有写入工具需要[策略审批](/zh/docs/reference/tools-policy)。省略 protocol 时按链自动选择（EVM 默认 Aave，Solana 默认 Kamino）。

## lend_tokens

向借贷协议供给代币。支持 Aave v3（EVM）、Compound v3（EVM）、Kamino（Solana）和 Marginfi（Solana）的原生执行。

### 参数

<TypeTable
  type={{
    wallet: {
      type: "string",
      description: "钱包名称。若省略，使用活跃钱包。",
      default: "active wallet",
    },
    account_index: {
      type: "integer",
      description: "钱包内的账户索引。若省略，使用活跃账户索引。",
      default: "active index",
    },
    chain: {
      type: "string",
      description: "执行的链：solana 或 EVM 链名称。",
      required: true,
    },
    protocol: {
      type: '"aave" | "compound" | "kamino" | "marginfi"',
      description: "协议选择。若省略，按链默认（EVM：aave；Solana：kamino）。",
    },
    token: {
      type: "string",
      description:
        "要供给的代币。EVM：ERC-20 合约地址或原生代币。Solana：SPL mint 公钥或原生代币。",
      required: true,
    },
    amount: {
      type: "string",
      description: "供给金额。注意：Aave/Compound 原生路径不支持 amount=max。",
      required: true,
    },
    amount_units: {
      type: '"ui" | "base"',
      description: "ui = 人类可读代币单位，base = 原始基础单位。",
      default: '"ui"',
    },
    usd_value: {
      type: "number",
      description: "用于策略执行的尽力美元价值。若省略，视为未知。",
    },
    usd_value_known: {
      type: "boolean",
      description: "美元价值追踪的覆盖标志。",
      default: "false",
    },
    pool_address: {
      type: "string",
      description: "Aave v3：可选 Pool 地址覆盖（适用于本地测试/模拟）。",
    },
    comet_address: {
      type: "string",
      description:
        "Compound v3：可选 Comet 市场地址覆盖。若省略，使用 USDC 市场的每链默认值。",
    },
    market: {
      type: "string",
      description:
        "Kamino：可选市场公钥覆盖（默认为 http.kamino_default_lend_market）。",
    },
    group: {
      type: "string",
      description:
        "Marginfi：可选 group 公钥覆盖（默认为 http.marginfi_default_group）。",
    },
  }}
/>

### 各协议行为

<Tabs
  items={[
    "Aave (EVM)",
    "Compound (EVM)",
    "Kamino (Solana)",
    "Marginfi (Solana)",
  ]}
>
  <Tab value="Aave (EVM)">
    - EVM 链的默认协议 - 原生执行：Seashail 直接调用 Aave v3 Pool -
    支持供给、提款、借款、还款 - 可用 `pool_address` 覆盖进行测试 -
    供给操作不支持 `amount=max`
  </Tab>
  <Tab value="Compound (EVM)">
    - 显式设置 `protocol: "compound"` - 通过 Compound v3 Comet 原生执行 -
    非默认市场可使用 `comet_address` 覆盖 - 使用 `token: "base"`
    供给基础代币（baseToken()）- 原生路径不支持 `amount=max`
  </Tab>
  <Tab value="Kamino (Solana)">
    - Solana 链的默认协议 - 通过 Kamino 借贷原生执行 - 非默认市场可使用 `market`
    覆盖 - 支持供给、提款、借款、还款
  </Tab>
  <Tab value="Marginfi (Solana)">
    - 显式设置 `protocol: "marginfi"` - 通过 Marginfi 原生执行 - 可用 `group`
    覆盖 - 若省略 `marginfi_account`，使用最近创建的账户
  </Tab>
</Tabs>

### 注意事项

- 需要[策略审批](/zh/docs/reference/tools-policy)——借贷操作可能被自动批准、需要确认或被阻止，取决于策略配置和美元价值。
- 协议按链自动选择：EVM 链默认 Aave，Solana 默认 Kamino。
- Aave/Compound 供给路径不支持 `amount=max`。
- 若未提供原生参数，回退到交易封装模式（EVM 需要 `to`/`data`，Solana 需要 `tx_b64`）。
- 另见：[DeFi 指南](/zh/docs/guides/defi)

- Aave 通过 `interest_rate_mode` 支持浮动和固定利率模式。
- 对于 Compound v3，`token` 必须为市场的基础代币。使用 `token="base"` 作为 baseToken() 的简写。
- 另见：[DeFi 指南](/zh/docs/guides/defi)
