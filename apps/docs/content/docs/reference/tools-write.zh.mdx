## request_airdrop

在 devnet、测试网或本地验证节点上请求 Solana 空投。在主网上被阻止。

### 参数

<TypeTable
  type={{
    wallet: {
      type: "string",
      description: "钱包名称。若省略，使用活跃钱包。",
      default: "active wallet",
    },
    account_index: {
      type: "integer",
      description: "钱包内的账户索引。若省略，使用活跃账户索引。",
      default: "active index",
    },
    chain: {
      type: '"solana"',
      description: "必须为 solana。",
      default: '"solana"',
    },
    address: {
      type: "string",
      description:
        "接收空投的 Solana 地址。若省略，使用选定账户的钱包 Solana 地址。",
    },
    amount: {
      type: "string",
      description: "请求的 SOL 数量。",
      required: true,
    },
    amount_units: {
      type: '"ui" | "base"',
      description: "ui = SOL，base = lamports。",
      default: '"ui"',
    },
  }}
/>

### 响应

```json
{
  "signature": "5UfD...txid",
  "amount": "1",
  "amount_units": "ui"
}
```

### 示例

```json
{
  "wallet": "my-wallet",
  "account_index": 0,
  "amount": "1",
  "amount_units": "ui"
}
```

### 注意事项

- 仅在 devnet、测试网和本地验证节点上可用。
- 若省略 `address`，Seashail 使用选定账户的钱包 Solana 地址。
- 速率限制可能因验证节点/水龙头而异。

## send_transaction

转移原生代币或同质化代币（Solana 上的 SPL；EVM 上的 ERC-20）。需要[策略审批](/zh/docs/guides/policy-and-approvals)。

      description: "钱包内的账户索引。若省略，使用活跃账户索引。",
      default: "active index",

    chain: {
      type: "string",

    },
    to: {
      type: "string",

    token: {
      type: "string",
      description:
        '代币标识符。原生代币使用 "native"（SOL/ETH 等），Solana SPL mint 公钥，或 EVM ERC-20 合约地址。',

      type: "string",

    wei 或原始基础单位 - 配置了诈骗地址黑名单时可能阻止接收方
  </Tab>
</Tabs>

### 注意事项

- 需要[策略审批](/zh/docs/guides/policy-and-approvals)——交易可能被自动批准、需要确认或被阻止，取决于策略配置和美元价值。
- 配置了诈骗地址黑名单时，Seashail 可能阻止接收方。
- Solana 上的 SPL 发送时，如有需要，Seashail 会自动创建接收方的关联代币账户。
- `wallet` 和 `account_index` 字段可选——若省略，Seashail 使用当前活跃的钱包和账户。

### 注意事项

- **原生路径需要三个参数全部提供：** `to_chain`、`token` 和 `amount` 必须全部提供，Seashail 才能执行原生 Wormhole/LayerZero 桥接。如果缺少任一参数，Seashail 将回退到交易封装模式。
- **不支持 `amount=max`**（Wormhole 原生路径）。
- **`redeem=true`（默认）**会尝试在目标链上自动赎回。这要求目标钱包有燃料（EVM 目标需要 ETH，Solana 目标需要 SOL）。
- **`bridge_provider`** 默认为 `"wormhole"`。EVM 链上的 OFT v1 代币也支持 LayerZero（`"layerzero"`）。
- **策略：** 跨链桥接必须在[策略配置](/zh/docs/guides/policy-and-approvals)中启用（`enable_bridge` 字段）。
- **状态追踪：** 使用 [`get_bridge_status`](/zh/docs/reference/tools-read#get_bridge_status) 以交易签名作为 `bridge_id` 来监控桥接进度、VAA 可用性和赎回状态。
