## 什么是 Pump.fun

Pump.fun 是一个基于 Solana 的平台，用于发行和交易 Meme 币。与传统代币发行不同，Pump.fun 代币遵循联合曲线机制：随着更多人买入，价格上涨；随着卖出，价格下跌。这创建了一种市场驱动的价格发现模型，供需直接实时影响代币价格。

Seashail 提供 4 个 Pump.fun 交互工具：

- `pumpfun_list_new_coins` — 发现最近发行的代币
- `pumpfun_get_coin_info` — 获取特定代币的详细信息
- `pumpfun_buy` — 用 SOL 买入 Pump.fun 代币
- `pumpfun_sell` — 将你的 Pump.fun 代币卖回 SOL

所有 Pump.fun 工具仅限 Solana。你必须有一个带 SOL 用于支付 Gas 手续费的 Solana 钱包。

## 发现新代币

使用 `pumpfun_list_new_coins` 查看最近的代币发行。此工具返回按发行时间排序的代币列表（最新的在前）。

响应包含：

- Mint 地址（用于买入/卖出）
- 代币名称和符号
- 描述
- 创建时间戳
- 联合曲线进度和市值

你可以配置结果数量（最多 200 个代币）。

这是只读操作——无需策略审批。

## 获取代币详情

使用 `pumpfun_get_coin_info` 并提供代币的 mint 地址，获取特定代币的详细信息。

响应包含：

- 名称、符号和描述
- 创建者地址
- 市值和联合曲线状态
- 虚拟 SOL 和代币储备
- 相关联合曲线账户地址

这些数据帮助你在交易前评估代币状态。只读操作——无需策略审批。

## 买入代币

使用 `pumpfun_buy` 购买 Pump.fun 代币。交易针对联合曲线智能合约执行。

必需参数：

- `mint` — 代币的 mint 地址（经过安全性验证）
- `amount_sol` — 你想花费的 SOL 数量

可选参数：

- `wallet` — 如果省略，使用你的活跃钱包
- `account_index` — 如果省略，使用你的活跃账户索引
- `usd_value` — 策略审批评估的可选覆盖值

买入操作受策略评估约束：

- 策略中的 `enable_swap` 必须为 `true`
- 交易价值会对照 `max_usd_per_tx` 和每日限额进行检查
- 分级审批适用（可能根据美元价值自动审批、需要确认或被拦截）

有关分级审批的工作方式，请参阅[策略与审批](/zh/docs/guides/policy-and-approvals)。

### 典型买入工作流程

1. 发现代币：`pumpfun_list_new_coins`
2. 调查特定代币：`pumpfun_get_coin_info`，使用 mint 地址
3. 买入：`pumpfun_buy`，使用 mint 和 SOL 数量
4. 如果提示则确认（通过 MCP 交互确认）

买入示例：

```json
{
  "mint": "ABC123...",
  "amount_sol": 0.5
}
```

这花费 0.5 SOL 以当前联合曲线价格买入 Pump.fun 代币。

### 联合曲线机制

联合曲线意味着：

- 你买入得越多，后续买家的价格就越高
- 大量买入会显著推高价格
- 价格影响取决于你的买入规模相对于曲线储备的大小

## 卖出代币

使用 `pumpfun_sell` 将你的 Pump.fun 代币卖回 SOL。

必需参数：

- `mint` — 代币的 mint 地址（经过验证）
- `percent` — 你持仓中要卖出的百分比（0-100）

使用 `percent: 100` 卖出全部仓位。

卖出操作受与买入相同的策略控制约束。

卖出示例：

```json
{
  "mint": "ABC123...",
  "percent": 100
}
```

这以当前联合曲线价格卖出该代币 100% 的持仓，收回 SOL。

## 风险注意事项

**Pump.fun 交易存在极高风险。只用你能承受完全损失的资金交易。**

### 波动性

Meme 币可能在几分钟内损失 90% 或更多的价值。价格在数秒内向任一方向波动 50% 以上是很常见的。联合曲线机制放大了波动性：大量卖出会导致价格暴跌，大量买入会导致价格飙升。

### 跑路风险

代币创建者可以随时抛售他们的持仓，通常是在初始炒作推高价格之后。这在 Meme 币领域极为普遍。

### 流动性风险

联合曲线代币可能流动性极低。大量卖出可能会大幅压低价格——你收到的 SOL 可能远少于预期。

### 无基本价值

大多数 Meme 币没有任何基础用途、收益或现实世界用例。价值完全由投机和市场情绪驱动。

### 策略保障措施

使用策略控制限制风险敞口：

- 设置保守的 `max_usd_per_tx` 限额
- 使用 `max_usd_per_day` 限制每日 Meme 币支出上限
- 考虑为实验性交易使用单独的钱包策略覆盖

有关配置详情，请参阅[策略与审批](/zh/docs/guides/policy-and-approvals)。
