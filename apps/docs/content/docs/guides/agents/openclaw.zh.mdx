title: OpenClaw
description: 通过 MCP stdio 将 Seashail 连接到 OpenClaw。
---

OpenClaw 可以通过 MCP stdio 连接到 Seashail（如果你的构建版本支持）。

> 推荐：安装 **Seashail OpenClaw 插件**以获得更完善的原生工具支持。

## 前置条件

- [Seashail 已安装](/zh/docs/getting-started/install)并在你的 `PATH` 中
- 已创建钱包（首次连接会自动创建默认钱包，也可手动调用 `create_wallet`）
- 支持 MCP 的 OpenClaw 构建版本

## 设置（推荐：OpenClaw 插件）

如果你已安装 OpenClaw，可以用一条命令完成所有设置：

```bash
seashail openclaw install
```

这会安装 Seashail OpenClaw 插件并在 `~/.openclaw/openclaw.json` 中启用它。

### 手动方式（OpenClaw CLI）

如果你更喜欢直接使用 OpenClaw CLI（例如在尚未将 `seashail` 添加到 PATH 的机器上）：

```bash
openclaw plugins install @seashail/seashail
openclaw plugins enable seashail
openclaw gateway restart
```

## 设置（备选：原生 MCP 支持）

如果你的 OpenClaw 构建版本直接支持 MCP stdio 服务器，你也可以配置一个名为 `seashail` 的 MCP 服务器：

- **命令：** `seashail`
- **参数：** `mcp`

## 设置（桥接/代理）

如果你的 OpenClaw 构建版本不原生支持 MCP，使用一个可以运行 stdio MCP 服务器并代理工具调用的 MCP 桥接工具。将桥接指向：

```bash
seashail mcp
```

如果你的桥接工具读取 `mcpServers` JSON 文件（通常命名为 `.mcp.json`），可以使用[任意 MCP 客户端（stdio）](/zh/docs/guides/agents/any-mcp-client) 中的配置格式。

## 测试网模式

- **命令：** `seashail`
- **参数：** `mcp --network testnet`

## 验证

设置完成后，让 OpenClaw 检查你的余额：

```
What's my SOL balance?
```

OpenClaw 应该通过 Seashail 调用 `get_balance` 工具。

## 故障排除

**不支持 MCP**
检查你的 OpenClaw 构建版本是否支持 MCP stdio。如果不支持，使用可以将工具调用转发到本地 stdio 进程的桥接/代理。

**"seashail: command not found"**
使用 Seashail 二进制文件的绝对路径。
